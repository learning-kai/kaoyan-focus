感谢你愿意改进 `考研专注`。本项目是本地优先的 Windows 学习工具，贡献时请优先保证用户数据安全、同步稳定性和学习模式状态机可靠性。

## 本地开发

```powershell
npm.cmd install
npm.cmd run dev
npm.cmd run tauri dev
```

常用检查：

```powershell
npm.cmd run lint
npm.cmd test
npm.cmd run build
```

## PR 要求

- 一个 PR 只做一类事情：功能、修复、重构、文档尽量分开。
- 不提交私钥、`.env*`、真实数据库、同步备份、日志、截图中的隐私数据或构建产物。
- 不随意改变 Tauri command 名称、SQLite 表结构、同步 payload 形状。
- 涉及数据库结构时，必须说明迁移方式、兼容策略和回滚风险。
- 涉及同步、飞书、备份恢复、学习模式状态机时，必须补充测试或明确人工验证步骤。
- 新增、删除、重命名核心文件后，同步更新 [项目文件结构说明.md](./项目文件结构说明.md)。

## 代码风格

- 前端遵循当前 React + TypeScript 写法，不引入大型状态库，除非有明确收益。
- Rust 后端优先保持命令边界清晰，Windows 能力集中在 `windows/`，数据库能力集中在 `storage/`。
- 文案和中文文档必须使用 UTF-8，乱码视为阻塞问题。
- 重构以降低认知负担为目标，不为拆而拆。

## Issue 建议

提交 bug 时请包含：

- 系统版本
- 应用版本
- 复现步骤
- 期望行为和实际行为
- 是否启用 WebDAV、对象存储、飞书、邮件提醒
- 可公开的截图或日志片段

不要粘贴真实密钥、token、邮箱授权码、飞书 App Secret 或完整数据库。
