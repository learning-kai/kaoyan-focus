# 项目文件结构说明

本文档列出当前桌面端仓库中需要长期维护的核心文件和目录职责。它的目标不是替代源码，而是让新贡献者和 AI 在进入项目时，能先建立一张稳定的结构地图。

不纳入本清单的忽略目录和文件：

- `.git/`
- `.claude/`
- `node_modules/`
- `dist/`
- `src-tauri/target/`
- `src-tauri/gen/`
- `*.log`
- `.env*`
- `src-tauri/updater/tauri-update.key`

更新规则：

- 新增、删除、重命名核心源码文件后，必须同步更新本文件。
- 文件职责发生明显变化时，必须同步更新对应说明。
- 本文件不保留“增量补充”章节；结构变化时直接修改对应条目。
- 公开仓库中不得记录真实密钥、本地数据库路径、个人账号或私有发布地址。

## 根目录

- `README.md`：面向用户和新贡献者的项目入口，说明项目价值、功能、安装开发、隐私边界、路线图和许可证。
- `LICENSE`：MIT 开源协议。
- `CONTRIBUTING.md`：普通贡献者开发、测试、提交 PR 的入口说明。
- `SECURITY.md`：安全边界、敏感数据、漏洞反馈方式。
- `CHANGELOG.md`：版本变更记录，发布时应记录用户可感知的改动。
- `FEATURES.md`：功能说明文档，记录当前产品能力、限制和验证清单。
- `技术路线.md`：早期技术路线和阶段规划，可作为历史设计背景阅读。
- `项目文件结构说明.md`：当前文件，维护仓库结构索引。
- `项目完整链路说明.md`：维护核心业务、数据、同步、发布链路。
- `项目开发规范（AI协作）.md`：维护 AI 和开发者协作时必须遵守的开发规则。
- `开发者AI开发与PR提交流程.md`：维护从本地开发到 PR 的标准流程。
- `package.json`：前端依赖、脚本、公开仓库元数据和版本号。
- `package-lock.json`：npm 锁定文件，保证依赖安装可复现。
- `tsconfig.json`：TypeScript 编译配置。
- `vite.config.ts`：Vite 构建配置。
- `index.html`：前端 HTML 入口。
- `.gitignore`：仓库忽略规则，必须持续忽略构建产物、日志、私钥和环境文件。
- `check-msvc-env.bat`：Windows MSVC 环境检查脚本。

## `.github/`

- `.github/pull_request_template.md`：PR 描述模板，要求说明改动、测试和风险。
- `.github/ISSUE_TEMPLATE/bug_report.md`：Bug 报告模板。
- `.github/ISSUE_TEMPLATE/feature_request.md`：功能建议模板。
- `.github/ISSUE_TEMPLATE/config.yml`：Issue 模板配置。

## `scripts/`

- `release-config.mjs`：发布配置解析与更新器 endpoint 更新逻辑。
- `prepare-release.mjs`：准备版本号、changelog、Tauri 配置和历史 Android 版本同步。桌面端公开发布时，Android 同步应视为维护者内部流程，后续建议改成显式 opt-in。
- `release-win.mjs`：Windows 安装包构建脚本。
- `release-win-update.mjs`：Windows 更新包、签名和 `latest.json` 生成脚本。
- `release-auto.mjs`：自动发布编排脚本，支持 dry run、构建和 GitHub Release 上传。
- `generate-latest-json.mjs`：生成 Tauri updater 所需的发布元数据。
- `tag-release.mjs`：版本 tag 辅助脚本。
- `release-base-url.txt`：默认更新资产下载基地址。公开仓库中应检查该地址是否符合当前发布仓库。

## `src/`

- `src/main.tsx`：React 应用入口。
- `src/App.tsx`：桌面端主页面装配、导航状态、自动同步、飞书自动同步、提醒检查和更新检查调度。
- `src/theme.ts`：主题初始化、应用和本地存储。
- `src/styles.css`：全局样式。新增 UI 时优先复用现有 class，避免无约束增长。

## `src/components/`

- `Layout.tsx`：主布局、侧边栏、顶部状态和主题切换。
- `ConfirmDialog.tsx`：统一确认弹窗。
- `TodayPlanDrawer.tsx`：今日计划抽屉。
- `ScheduleDrawer.tsx`：课表块编辑抽屉。

## `src/pages/`

- `FocusPage.tsx`：专注页，承接学习模式启动、运行态展示、科目切换、主设备控制和学习中刷新。
- `AlarmPage.tsx`：一次性闹钟页面。
- `ChecklistPage.tsx`：清单和今日任务页面。
- `SchedulePage.tsx`：课表页面。
- `ReviewPage.tsx`：每日复盘和周复盘页面。
- `WhitelistPage.tsx`：软件、网站、PotPlayer 白名单页面。
- `StatsPage.tsx`：学习统计页面。
- `SettingsPage.tsx`：设置页组合层，负责跨面板状态、初始化、刷新和保存协调。

## `src/pages/settings/`

- `BasicSettingsPanel.tsx`：基础学习参数、主题和提醒音设置。
- `SyncSettingsPanel.tsx`：WebDAV、对象存储、同步日志、备份预检和恢复。
- `IntegrationsPanel.tsx`：飞书和邮件提醒配置。
- `SystemPanel.tsx`：强制规则、更新、前台检测、数据目录和运行健康。
- `SettingsPrimitives.tsx`：设置页通用小组件。
- `types.ts`：设置页内部展示状态类型。

## `src/services/`

- `tauriInvoke.ts`：Tauri 命令调用封装和浏览器预览兜底。
- `tauriEvents.ts`：Tauri 事件监听封装。
- `settingsApi.ts`：应用基础设置、提醒音、本地数据目录、同步设备 ID。
- `syncApi.ts`：WebDAV、对象存储、同步运行记录、备份预检和恢复。
- `feishuApi.ts`：飞书设置、OAuth、手动同步、重建和运行记录。
- `emailApi.ts`：邮件提醒设置、测试发送、到期任务检查。
- `healthApi.ts`：运行健康状态。
- `focusApi.ts`、`checklistApi.ts`、`scheduleApi.ts`、`reviewApi.ts`、`whitelistApi.ts`、`alarmApi.ts`、`monitorApi.ts`：各业务页面对应的前后端 API 边界。
- `systemApi.ts`：桌面系统能力，如外部链接、通知、前台应用、数据目录和更新安装。
- `updateApi.ts`：更新检查和安装流程。
- `alertApi.ts`：桌面提醒和提醒音播放。
- `studyModeLock.ts`：学习模式运行时配置锁定判断。

## `src/types/`

- `settings.ts`：应用设置、同步设置、飞书、邮件、运行健康等类型。
- `focus.ts`、`checklist.ts`、`schedule.ts`、`review.ts`、`whitelist.ts`、`alarm.ts`、`monitor.ts`、`navigation.ts`：各业务域前端类型定义。

## `src-tauri/src/`

- `main.rs`：Tauri 二进制入口。
- `lib.rs`：Tauri builder、插件、托盘、窗口事件、命令注册和 setup 装配。
- `background_tasks.rs`：后台任务启动入口，统一管理学习模式 tick、对象存储轮询和同步备份清理。
- `credential.rs`：本机凭据读写封装。
- `runtime_health.rs`：运行健康状态聚合。
- `dashboard_server.rs`：学习数据仪表盘服务。

## `src-tauri/src/storage/`

- `db.rs`：SQLite 打开、建表、兼容迁移、索引和默认数据初始化。
- `mod.rs`：存储模块出口。

## `src-tauri/src/focus/`

- `session.rs`：专注记录模型。
- `subject.rs`：科目模型。
- `mod.rs`：专注领域模型出口。

## `src-tauri/src/windows/`

- `foreground.rs`：Windows 前台窗口识别。
- `browser_url.rs`：浏览器 URL 识别。
- `running_processes.rs`：运行进程枚举。
- `window_control.rs`：窗口控制能力。
- `potplayer.rs`：PotPlayer 当前播放信息识别。
- `mod.rs`：Windows 能力出口。

## `src-tauri/src/whitelist/`

- `app.rs`：白名单应用模型。
- `matcher.rs`：白名单匹配规则和测试。
- `mod.rs`：白名单领域出口。

## `src-tauri/src/commands/`

- `alarm.rs`：闹钟命令。
- `checklist.rs`：清单、今日任务和清单变更触发同步。
- `email.rs`：邮件提醒配置、SMTP 测试、到期任务邮件发送。
- `health.rs`：运行健康命令。
- `monitor.rs`：前台检测、白名单执行、自动科目切换。
- `review.rs`：每日复盘和周复盘。
- `schedule.rs`：课表模板和课表块。
- `settings.rs`：应用基础设置、提醒音和数据目录。
- `whitelist.rs`：软件、网站、PotPlayer 白名单命令。

## `src-tauri/src/commands/focus/`

- `models.rs`：学习模式和统计相关结构体、常量。
- `study_mode.rs`：学习模式启动、暂停、恢复、休息确认、结束、重置和后台 tick。
- `sessions.rs`：普通专注记录增删改查和恢复。
- `state.rs`：学习模式状态推进、持久化读写和运行态同步。
- `stats.rs`：学习统计。
- `subjects.rs`：科目列表。
- `sync_trigger.rs`：学习模式变更后的对象存储和飞书同步触发。
- `helpers.rs`：时间、数据库路径、记录查询和通用校验。
- `mod.rs`：目录模块装配。对外路径必须继续保持 `commands::focus::xxx`。

## `src-tauri/src/commands/sync/`

- `models.rs`：同步设置、状态、运行记录、备份、R2 v3 数据结构。
- `webdav.rs`：WebDAV 设置、连接测试、上传、下载和自动同步命令。
- `object_storage.rs`：对象存储设置、手动上传/下载、自动同步和轮询入口。
- `object_storage_protocol.rs`：R2/S3 v3 操作日志协议、manifest、snapshot、lock 和远端状态加载。
- `backup.rs`：本地和远端备份、预检、恢复、清理。
- `runs.rs`：同步运行记录查询命令。
- `logging.rs`：同步结果记录、详情构建和验证报告。
- `webdav_helpers.rs`：WebDAV 请求、远端路径、数据库校验和通用辅助函数。
- `mod.rs`：目录模块装配。对外路径必须继续保持 `commands::sync::xxx`。

## `src-tauri/src/commands/feishu/`

- `settings.rs`：飞书设置、状态、OAuth 启动和退出登录命令。
- `sync.rs`：飞书同步命令、变更触发和同步锁。
- `client.rs`：飞书 HTTP client。
- `tasks.rs`：飞书任务清单、任务同步和本地任务映射。
- `calendar.rs`：飞书日历事件同步。
- `links.rs`：本地实体、sync_meta、飞书 link 的加载和写入。
- `auth.rs`：OAuth 回调、token 刷新、凭据持久化。
- `runs.rs`：飞书同步运行记录。
- `rebuild.rs`：从本地重建飞书任务清单和备份。
- `tests.rs`：飞书同步关键行为测试。
- `utils.rs`：URL、时间、marker、设置读写等辅助函数。
- `mod.rs`：目录模块装配。对外路径必须继续保持 `commands::feishu::xxx`。

## `src-tauri/src/sync_package/`

- `models.rs`：跨设备共享 payload 和实体模型。
- `export.rs`：从本地 SQLite 导出共享同步 payload。
- `import.rs`：将共享 payload 导入本地 SQLite。
- `merge.rs`：共享 payload 合并入口。
- `identity.rs`：sync_id、active study merge、本地实体版本和冲突处理。
- `tests.rs`：同步包关键行为测试。
- `mod.rs`：目录模块装配。对外继续暴露原有函数名。

## 维护提醒

- `src-tauri/src/sync_package/identity.rs` 和 `src-tauri/src/commands/sync/object_storage_protocol.rs` 仍偏长，是下一轮低风险深拆候选。
- `src/styles.css` 是全局样式集中点，新增页面前应优先复用现有结构，避免样式失控。
- 任何改变 Tauri command、SQLite 表结构、同步 payload 的改动，都必须同步更新链路文档和测试说明。
