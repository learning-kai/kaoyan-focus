# 考研专注功能说明

当前版本：`1.9.2`

`考研专注` 是一个面向 Windows 的本地学习约束工具，核心目标是在一段学习模式时间内强制执行番茄钟、休息流程和前台规则，减少非学习应用对考研复习的干扰。应用使用 Tauri 2 + React + TypeScript + Rust 构建，数据存储在本机 SQLite 数据库中。

## 1. 主界面与导航

应用包含八个主页面：

- `专注`（`Alt+1`）：沉浸式学习准备台和极简番茄钟，负责启动学习模式、展示番茄钟状态、休息状态和前台规则执行状态。
- `清单`（`Alt+2`）：管理今日真正要完成的任务，并可把任务推进到课表。
- `课表`（`Alt+3`）：把今日任务和手动安排落到一天时间轴，支持新增、拖拽、调整和从课表启动学习。
- `白名单`（`Alt+4`）：管理学习阶段的软件、网站和 PotPlayer 视频规则，支持白名单和黑名单两种模式。
- `复盘`（`Alt+5`）：记录每日总结、明日重点和周复盘。
- `统计`（`Alt+6`）：查看学习时长、科目分布和干扰排行。
- `闹钟`（`Alt+7`）：配置全局一次性提醒。
- `设置`（`Alt+8`）：配置默认学习参数、主题、提醒、同步、集成、数据路径、前台应用检测和在线更新。

主导航支持鼠标点击、URL hash 直达和 `Alt+数字` 键盘切换；当输入框、下拉框、文本域或确认弹窗处于操作中时，全局快捷键不会抢走当前编辑流程。

窗口默认尺寸为 `1100 x 760`，最小尺寸为 `960 x 680`。应用带系统托盘图标，托盘菜单包含“显示主界面”和“退出”。

## 2. 学习模式控制台

学习模式是当前应用的核心流程。它不是单个倒计时，而是在一个总学习时长内循环执行番茄钟和休息。

### 2.1 可配置项

在专注页开始前可以配置：

- 学习模式总时长，默认 `120` 分钟。
- 单轮番茄钟时长，默认 `25` 分钟。
- 短休息时长，默认 `5` 分钟。
- 长休息时长，默认 `15` 分钟。
- 长休息间隔，默认每 `4` 个番茄钟进入一次长休息。
- 科目，可选择政治、英语、数学、专业课，或不指定科目。
- 专注模式：
  - 普通模式：常规学习约束。
  - 强制模式：保持更严格的学习约束，不提供手动提前结束入口。

专注页提供快捷预设：

- 学习模式时长：`60m`、`120m`、`180m`、`240m`
- 番茄钟时长：`25m`、`45m`、`60m`、`90m`
- 短休息时长：`5m`、`10m`、`15m`、`20m`
- 长休息时长：`10m`、`15m`、`20m`、`30m`
- 长休息间隔：每 `2`、`3`、`4`、`6` 轮一次

### 2.2 阶段状态

学习模式包含以下阶段：

- `idle`：待开始。
- `focus`：番茄钟专注中。
- `awaiting_break`：本轮番茄钟已到点，等待本人确认开始休息。
- `break`：休息中。
- `finished`：学习模式已完成。
- `emergency_exited`：历史兼容状态，当前界面不再提供应急退出入口。

界面会显示：

- 未开始时显示完整学习计划、前台规则监控、今日统计和最近记录。
- 学习模式运行中切换为极简番茄钟，只保留当前阶段、轮次、倒计时、总进度、阶段进度、科目和前台规则执行状态。
- 运行中不显示学习计划编辑、历史列表、“结束学习模式”或“应急退出”入口。
- 设置页和白名单页在学习模式运行中会锁定全部配置改动，只允许查看状态。

### 2.3 番茄钟和休息规则

学习模式启动后，后端会创建当前番茄钟专注记录，并进入 `focus` 阶段。

番茄钟到点后不会自动进入休息，而是进入 `awaiting_break`：

- 用户必须本人点击“确认开始休息”。
- 未确认休息前，学习时间继续累计。
- 未确认休息前，前台规则强制执行继续生效。
- 确认休息后，当前番茄钟记录会写入完成状态。
- 如果当前轮次能被长休息间隔整除，则确认后进入长休息；否则进入短休息。

进入 `break` 后：

- 应用按照当前轮次对应的短休息或长休息时长倒计时。
- 休息阶段暂停前台规则强制关闭逻辑。
- 休息结束后，后端自动进入下一轮番茄钟。
- 下一轮番茄钟会创建新的专注记录。

当学习模式总时长到达后：

- 当前运行中的番茄钟会结束并写入实际学习时长。
- 学习模式进入 `finished`。
- 后台强制执行停止。

### 2.4 后台运行

学习模式由 Rust 后端主控，不依赖前端页面的 `setInterval` 维持业务状态。

应用启动后会创建一个后台循环：

- 每 `3` 秒推进学习模式状态。
- 每 `3` 秒在需要强制执行时检查当前前台窗口。
- 窗口隐藏到托盘后仍继续运行。
- 应用重启后会从 SQLite 中恢复活动中的学习模式状态。

这意味着：

- 关闭主窗口不会打断学习模式。
- 关闭主窗口时会隐藏到托盘，而不是退出应用。
- 托盘左键点击可以重新显示主界面。
- 活动学习模式期间，托盘菜单的“退出”不会真正退出，而是重新显示主界面。

### 2.5 中断策略

当前版本不再使用“超时自动中断”作为主流程。

具体规则：

- 应用重启、窗口隐藏、番茄钟超时都不会自动写入 `interrupted_after_due`。
- 学习模式会根据数据库中的时间戳恢复或推进。
- 历史中已有的 `interrupted` 记录仍可显示。
- 新学习模式流程不提供手动提前结束或应急退出入口，只能按总学习时长自然完成。

### 2.6 提前结束策略

当前版本关闭了“结束学习模式”和“应急退出”的用户入口：

- 学习模式启动后，专注页进入极简番茄钟，只保留刷新状态和确认休息等必要流程按钮。
- 活动学习模式期间关闭窗口会进入托盘，不会退出应用。
- 活动学习模式期间托盘“退出”会被阻止，并重新显示主界面。
- 如需停止学习模式，需要等待总学习时长自然结束。
- 后端仍保留历史兼容字段和旧状态显示，但当前界面不暴露应急退出按钮。

### 2.7 声音与桌面提醒

以下真实阶段变化会播放一声提示音，并在 Windows 右下角显示桌面通知：

- 学习模式开始。
- 番茄钟到点并进入等待休息确认。
- 本人确认开始休息。
- 休息结束并进入下一轮番茄钟。
- 学习模式自然完成。

切换页面后再次进入专注页不会触发通知，通知只绑定真实阶段变化。

## 3. 前台规则强制执行

学习阶段和等待休息确认阶段会启用前台规则强制执行。规则条目仍复用 `whitelist_apps`，但可以在设置页选择两种语义：

- 白名单模式：默认拦截，只放行命中规则的软件、网站或 PotPlayer 视频。
- 黑名单模式：默认放行，只拦截命中规则的软件、网站或 PotPlayer 视频。

### 3.1 执行方式

后端通过 Windows API 读取当前前台窗口：

- 进程 ID。
- 进程名。
- 进程路径。
- 窗口标题。

然后根据当前规则模式判断是否允许。如果不允许：

- 应用会向该窗口发送 `WM_CLOSE`。
- 记录一次 `blocked_foreground_detected` 干扰事件。
- 当前专注记录的 `interruption_count` 增加。
- 前端显示本次检测结果、进程名、窗口标题、识别原因和累计拦截次数。

为了避免同一个前台进程连续重复刷屏，后端会记住最近一次被拦截的 `(session_id, process_name)`，相同进程连续出现时不会重复写入干扰事件。

### 3.2 默认放行进程

以下进程默认放行，不需要手动加入白名单：

- `explorer.exe`
- `sihost.exe`
- `ctfmon.exe`
- `textinputhost.exe`
- `applicationframehost.exe`
- `kaoyan-focus.exe`

这些进程通常是 Windows 桌面、输入法、系统窗口或本应用本身。

### 3.3 软件规则

软件规则基于进程名匹配。白名单模式下命中规则表示放行；黑名单模式下命中规则表示拦截。

用户可以：

- 手动输入软件名称和进程名。
- 从当前运行进程中选择。
- 从最近干扰记录中一键加入。
- 填写可选备注。
- 保存进程路径。
- 启用或停用某条规则。
- 删除某条规则。

重复添加同一进程名时，不会创建重复记录，而是更新已有记录并重新启用。

### 3.4 网站规则

网站规则基于域名匹配。白名单模式下命中规则表示放行；黑名单模式下命中规则表示拦截。

用户可以添加：

- 网站名称，例如“中国大学 MOOC”。
- 域名，例如 `icourse163.org`。
- 可选备注。

域名会被规范化：

- 去掉 `http://`
- 去掉 `https://`
- 去掉 `www.`
- 去掉末尾 `/`
- 转为小写

匹配规则支持：

- 精确域名匹配，例如 `icourse163.org`
- 子域名匹配，例如 `www.icourse163.org`、`study.icourse163.org`

### 3.5 PotPlayer 视频规则

PotPlayer 视频规则只在前台进程是 `PotPlayer` 时生效，并且优先级高于普通软件规则。

用户可以添加：

- 单个视频文件规则。
- 整个视频目录规则。
- 从当前 PotPlayer 正在播放的视频一键加入。
- 手动填写完整文件路径或目录路径。
- 可选备注。

匹配规则支持：

- `potplayer_video_file`：当前播放视频完整路径精确匹配。
- `potplayer_video_directory`：当前播放视频位于允许目录下。

识别逻辑：

- 优先从 PotPlayer 安装目录下的 `Playlist/*.dpl` 读取当前播放路径。
- 如果 `playname` 滞后，会用窗口标题里的文件名对照播放列表条目。
- 当前未识别到播放视频时，按空播放器放行，不自动拦截。

### 3.6 浏览器识别范围

当前网站规则通过浏览器窗口标题和可选浏览器 URL 尝试识别域名。支持的浏览器进程包括：

- `chrome.exe`
- `msedge.exe`
- `firefox.exe`
- `brave.exe`
- `opera.exe`
- `vivaldi.exe`
- `iexplore.exe`

限制：

- 该方案依赖窗口标题中是否出现域名。
- 某些网站或浏览器主题可能不在标题中显示域名。
- 当前版本没有浏览器扩展或网络层 URL 读取能力。
- 因此网站规则是可用的轻量方案，但精度不如浏览器扩展方案。

### 3.7 规则页面优化

规则页面已从垂直堆叠布局改为选项卡切换布局：
- **查看规则选项卡**：默认显示，展示当前所有规则
- **添加规则选项卡**：点击后显示添加表单和快速来源面板
- **模式自适应文案**：白名单模式显示“放行”，黑名单模式显示“拦截”
- **减少滚动**：页面长度减少约50%，提升用户体验
- **响应式设计**：在移动设备上自动适配布局

## 4. 干扰记录

当学习阶段检测到被前台规则拦截的窗口时，会写入干扰事件。

记录内容包括：

- 关联的专注记录 ID。
- 进程名。
- 进程路径。
- 窗口标题。
- 事件类型。
- 采取的动作。
- 创建时间。

当前事件类型主要是：

- `blocked_foreground_detected`

动作包括：

- `close_window`
- `close_window_failed`

这些记录会用于：

- 专注页展示当前累计拦截次数。
- 白名单页展示最近干扰记录，方便一键加入当前模式对应的规则。
- 统计页展示干扰排行。

## 5. 学习统计

统计页按照本地专注记录汇总学习数据。

### 5.1 时间维度

当前支持：

- 今日学习时长。
- 本周学习时长。
- 本月学习时长。
- 累计干扰次数。

学习时长只统计状态为 `finished` 的专注记录。

### 5.2 科目维度

默认科目包括：

- 政治
- 英语
- 数学
- 专业课

统计页会展示每个启用科目的累计学习时长。科目数据保存在本地数据库中，首次初始化数据库时自动写入默认科目。

### 5.3 干扰排行

统计页会按被拦截次数展示最常干扰学习的进程。

每条干扰排行包含：

- 进程名。
- 最近一次窗口标题。
- 进程路径。
- 最近一次干扰时间。
- 累计干扰次数。

## 6. 设置

设置页负责保存默认行为和辅助诊断。

### 6.1 默认学习参数

可配置：

- 默认学习模式时长，范围 `1` 到 `720` 分钟。
- 默认番茄专注时长，范围 `1` 到 `120` 分钟。
- 默认短休息时长，范围 `1` 到 `60` 分钟。
- 默认长休息时长，范围 `1` 到 `120` 分钟。
- 默认长休息间隔，范围 `1` 到 `12` 轮。
- 默认专注模式：普通或强制。
- 前台规则模式：白名单或黑名单。

设置会保存到 SQLite 的 `settings` 表中。进入专注页时会自动读取这些默认值。

学习模式处于 `focus`、`awaiting_break` 或 `break` 阶段时，设置页全部配置入口会禁用，包括默认学习参数、WebDAV 配置、手动上传/下载和更新安装操作。

### 6.2 WebDAV 数据同步

设置页可填写 WebDAV 地址、用户名、密码或应用密码，以及远端数据库路径。应用支持：

- 保存 WebDAV 配置。
- 测试 WebDAV 连接。
- 手动上传本机 SQLite 数据库。
- 手动从云端恢复数据库，恢复前会备份本地旧库并校验远端 SQLite 完整性。
- 应用启动 `5` 秒后自动同步一次，未配置 WebDAV 时静默跳过。

自动同步采用时间保护策略：

- 如果远端文件不存在，上传本地数据库。
- 如果远端 `Last-Modified` 新于本地数据库修改时间，先下载、校验、备份本地数据库，再回传同步。
- 如果本地数据库更新，上传到 WebDAV。
- 如果学习模式正在运行，或远端无法提供可靠修改时间，则跳过自动同步，避免误覆盖。

### 6.3 飞书任务与日历同步

飞书集成用于把本地清单任务和课表同步到飞书任务清单与日历。

当前桌面端采用本地可落地的双向同步策略：

- 本地修改后立即触发同步，把本地变更推到飞书。
- 自动任务会周期性拉取飞书任务和日历，作为没有公网事件接收能力时的兜底。
- link 表保存本地实体、远端 ID、父容器 ID、远端最后修改时间和内容指纹。
- 任务同步使用本地任务指纹与远端任务指纹判断“哪一边真的改了”，避免只因为飞书 `updated_at` 更新就覆盖本地改动。
- 双方都改过时才进入冲突仲裁：明显更新的一侧胜出；本地触发的同步在时间接近时优先保留本地修改。

推荐的实时增强方向是接入飞书事件订阅：官方支持事件订阅、长连接接收事件、Webhook 回调，以及 `task.task.update_user_access_v2` 任务更新事件。桌面端没有公网地址时，不能只依赖 Webhook；更合理的是“长连接/Webhook 能接就实时接，接不到就继续用短周期增量轮询兜底”。

### 6.4 本地数据位置

设置页会展示当前 SQLite 数据库路径：

- 数据目录来自 Tauri 的 `app_data_dir`。
- 数据库文件名为 `kaoyan-focus.sqlite3`。

### 6.5 当前前台应用检测

设置页提供“检测当前应用”按钮，用于验证 Windows 前台窗口识别是否正常。

检测结果包括：

- 进程名。
- 进程 ID。
- 窗口标题。
- 进程路径。

这个功能适合用于确认某个软件应该加入规则时的进程名。

### 6.6 状态提示

设置页展示以下只读能力状态：

- 专注期间关闭窗口时最小化到托盘。
- 记录前台规则拦截事件。

## 7. 在线更新

应用集成了 Tauri updater。

### 7.1 用户侧功能

设置页包含在线更新模块：

- 检查更新。
- 显示发现的新版本号和更新说明。
- 下载并安装更新。
- 显示下载进度。
- 下载完成后通过 Tauri process 插件重启应用。

### 7.2 更新配置

当前 Tauri 配置中启用了 updater：

- 使用内置公钥校验更新包签名。
- 更新地址为 `https://github.com/learning-kai/kaoyan-focus/releases/latest/download/latest.json`。
- Windows 安装模式为 `passive`。

注意：当前 endpoint 指向公开仓库 `learning-kai/kaoyan-focus` 的 latest release 下载目录，发版时需要把安装包、签名和 `latest.json` 一起上传到该 Release。

### 7.3 发布元数据

项目包含 `scripts/generate-latest-json.mjs`：

- 读取安装包对应的 `.sig` 签名文件。
- 读取 `package.json` 中的版本号。
- 生成 Tauri updater 需要的 `latest.json`。
- 输出到安装包所在目录。

`latest.json` 内容包含：

- 版本号。
- 更新说明。
- 发布时间。
- Windows x64 平台下载 URL。
- 安装包签名。

## 8. Windows 发布包

当前项目面向 Windows 发布。

### 8.1 打包方式

使用 Tauri NSIS 打包，生成 `.exe` 安装程序。

配置位置：

- `src-tauri/tauri.conf.json`

当前配置：

- `productName`：`考研专注`
- `version`：以 `package.json`、`src-tauri/Cargo.toml` 和 `src-tauri/tauri.conf.json` 中的当前版本为准。
- `identifier`：`com.kaoyan.focus`
- `bundle.targets`：`nsis`
- `bundle.icon`：`icons/icon.ico`

### 8.2 发布脚本

`package.json` 中包含：

- `npm.cmd run build`：执行 TypeScript 检查和 Vite 构建。
- `npm.cmd run tauri`：运行 Tauri CLI。
- `npm.cmd run release:win`：执行 NSIS 打包。
- `npm.cmd run release:win:update`：打包、签名并生成更新元数据。
- `npm.cmd run release:metadata`：基于已有安装包和签名生成 `latest.json`。

`release-win-update.mjs` 会根据 `package.json` 的版本号自动拼出安装包路径，例如：

```text
src-tauri/target/release/bundle/nsis/kaoyan-focus_<version>_x64-setup.exe
```

### 8.3 签名状态

当前支持 Tauri updater 的更新签名，但不包含 Windows 代码签名证书。

因此：

- 安装包是未做 Windows Authenticode 代码签名的测试版。
- updater 使用 Tauri/minisign 私钥生成 `.sig` 进行更新包校验。
- 私钥文件应保留在本机并被 `.gitignore` 忽略。
- 公钥写入 `tauri.conf.json`。

## 9. 托盘与窗口行为

应用启动后会创建系统托盘图标。

托盘能力：

- 左键点击托盘图标显示主界面。
- 托盘菜单“显示主界面”显示并聚焦窗口。
- 托盘菜单“退出”在无活动学习模式时退出应用。
- 活动学习模式或活动专注记录存在时，“退出”会被阻止，并显示主界面。

窗口关闭行为：

- 没有活动学习模式时，关闭窗口会正常退出。
- 有活动学习模式或活动专注记录时，关闭窗口会隐藏到托盘。
- 隐藏到托盘后，后台学习模式和前台规则强制执行继续运行。

## 10. 本地数据存储

应用使用 SQLite 本地数据库，文件名为：

```text
kaoyan-focus.sqlite3
```

数据库自动创建，表结构通过启动时迁移保证存在。

### 10.1 数据表

当前主要表：

- `focus_sessions`：每轮番茄钟或单次专注记录。
- `study_modes`：学习模式总状态和当前阶段。
- `whitelist_apps`：软件、网站和 PotPlayer 视频规则，白名单/黑名单模式共用。
- `app_events`：前台规则拦截事件。
- `subjects`：科目。
- `settings`：用户设置。

### 10.2 focus_sessions

记录每轮专注：

- 模式：普通或严格。
- 科目 ID。
- 计划时长。
- 实际时长。
- 开始时间。
- 结束时间。
- 状态。
- 结束原因。
- 干扰次数。
- 应急退出次数，历史兼容字段，当前界面不再提供应急退出入口。
- 创建和更新时间。

状态包括：

- `running`
- `finished`
- `interrupted`
- `emergency_exited`，历史兼容状态

### 10.3 study_modes

记录一次完整学习模式：

- 总学习时长。
- 番茄钟时长。
- 短休息时长。
- 长休息时长。
- 长休息间隔。
- 当前阶段。
- 当前轮次。
- 学习开始时间。
- 当前阶段开始时间。
- 结束时间。
- 当前专注记录 ID。
- 状态。
- 完成原因。

### 10.4 whitelist_apps

软件、网站和 PotPlayer 视频规则共用一张表，通过 `match_type` 区分：

- `process_name`：软件进程名。
- `website_domain`：网站域名。
- `potplayer_video_file`：PotPlayer 单个视频文件。
- `potplayer_video_directory`：PotPlayer 视频目录。

字段包括：

- 名称。
- 进程名或域名。
- 路径。
- 备注。
- 是否启用。
- 创建和更新时间。

### 10.5 app_events

记录干扰事件：

- 关联专注记录。
- 进程名。
- 进程路径。
- 窗口标题。
- 事件类型。
- 执行动作。
- 创建时间。

### 10.6 subjects

首次启动会写入默认科目：

- 政治
- 英语
- 数学
- 专业课

每个科目包含名称、颜色、启用状态和时间戳。

### 10.7 settings

以 key-value 形式保存设置项：

- `default_study_minutes`
- `default_focus_minutes`
- `break_minutes`
- `default_focus_mode`
- `whitelist_mode`
- `emergency_cooldown_seconds`，历史兼容设置，当前界面不再展示

## 11. 技术能力边界

### 11.1 已实现能力

- Windows 本地桌面应用。
- 后台学习模式状态机。
- 后台番茄钟和休息循环。
- 本人确认后才开始休息。
- 休息结束自动进入下一轮番茄钟。
- 学习总时长结束自动完成。
- 关闭手动结束学习模式入口。
- 关闭应急退出入口。
- 托盘隐藏和恢复。
- 活动学习期间阻止直接退出。
- Windows 前台窗口识别。
- 前台规则命中的窗口自动关闭。
- 软件白名单和黑名单模式。
- 网站白名单和黑名单模式。
- 最近干扰一键加入当前规则模式。
- 学习统计。
- 科目统计。
- 干扰排行。
- 本地 SQLite 持久化。
- 设置持久化。
- WebDAV 手动同步和启动自动同步。
- 在线更新 UI。
- NSIS Windows 安装包。
- 自定义应用图标。
- 更新包签名元数据生成。

### 11.2 当前限制

- 网站规则依赖浏览器窗口标题或可选 URL 识别域名，不是浏览器扩展级精度。
- 前台规则关闭使用 `WM_CLOSE`，如果目标软件阻止关闭或需要管理员权限，可能关闭失败。
- 当前不包含 Windows Authenticode 代码签名。
- 当前不包含 macOS/Linux 发布配置。
- 当前不包含开机自启。
- WebDAV 同步是整库上传/下载，不做逐条记录合并。
- 当前 updater endpoint 使用 `https://github.com/learning-kai/kaoyan-focus/releases/latest/download/latest.json`，发布资产需要随 GitHub Release 一起维护。

## 12. 推荐验证清单

每次发版前建议验证：

- `cargo check`
- `npm.cmd run build`
- 启动学习模式后关闭窗口，确认应用进入托盘且后台继续运行。
- 番茄钟到点后进入等待休息确认。
- 不确认休息时，学习时间继续累计且前台规则强制执行继续生效。
- 确认休息后进入休息倒计时。
- 休息结束后自动进入下一轮番茄钟。
- 学习总时长到点后自动完成。
- 活动学习模式期间不显示“结束学习模式”和“应急退出”入口。
- 活动学习模式期间设置页和规则页配置入口不可编辑。
- 从其他页面切回专注页不会误触发右下角通知或提示音。
- 被前台规则拦截的软件会被关闭并写入干扰记录。
- 软件规则能添加、启用、停用、删除，并按白名单/黑名单模式切换语义。
- 网站规则能添加并在支持的浏览器标题或 URL 中命中。
- 统计页能显示今日、本周、本月、科目和干扰排行。
- 设置页能保存默认参数。
- 应用启动 `5` 秒后能按 WebDAV 配置自动同步，未配置时无弹窗和报错。
- 在线更新能检查到正确的 `latest.json`。
- NSIS 安装包能生成、安装、启动。
